# AI-Native Workflow Kit

This repository is a Markdown operating manual for delegating software work to AI coding agents while keeping human approval, review, and final product judgment explicit.

It is for a project builder who works across C, Java, WebGL, databases, embedded systems, desktop apps, educational tools, and portfolio projects. The kit helps the human product owner define work, choose an autonomy level, prepare context, delegate to an agent, review the diff, and improve the system after repeated failures.

This is not an app and not a prompt quote collection. Use it as a working folder of decisions, briefs, review gates, and retrospectives:

- [START_HERE.md](START_HERE.md) for the day-to-day workflow.
- [docs/](docs/) for the operating model, autonomy levels, context, permissions, review, and maintenance rules.
- [workflows/](workflows/) for repeatable task patterns.
- [templates/](templates/) for briefs, plans, review packets, decisions, and retrospectives.
- [checklists/](checklists/) for fast human review gates.
- [examples/](examples/) for realistic tasks from the owner's domains.
- [adapters/](adapters/) for Codex, Claude Code, GitHub Copilot, and Cursor.
- [research/](research/) for source-backed claims and caveats.
- [evals/](evals/) for improving the workflow with small local golden tasks.
- [decisions/](decisions/) and [failures/](failures/) for learning over time.

## How To Use This Tomorrow

1. Open [START_HERE.md](START_HERE.md).
2. Pick a workflow from [workflows/choose-a-workflow.md](workflows/choose-a-workflow.md).
3. Fill [templates/agent-task-brief.md](templates/agent-task-brief.md).
4. Choose an autonomy level from [docs/autonomy-levels.md](docs/autonomy-levels.md).
5. Give the brief and context packet to the agent.
6. Review the output with the relevant checklist.
7. Save the review packet, note repeated failures, and update the kit only when the lesson is reusable.
