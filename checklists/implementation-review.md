# Implementation Review

Use while inspecting the diff. Start with behavior and risk before style.

- [ ] Diff matches the approved task.
- [ ] No unrelated refactors, formatting churn, or opportunistic cleanup.
- [ ] Existing patterns are followed.
- [ ] Edge cases from the brief are handled or documented.
- [ ] Agent did not invent files, APIs, commands, or tool capabilities.
- [ ] Tests or manual evidence support the changed behavior.
- [ ] Review packet points the human to the riskiest lines or decisions.
