# Overengineering Review

Use when a change feels bigger than the task.

- [ ] The simplest workflow that fits the risk was used.
- [ ] No abstraction was added without a second concrete use.
- [ ] No multi-agent process was used where one agent was enough.
- [ ] No dependency, framework, or tool was added without approval.
- [ ] No broad refactor is bundled with feature or bug work.
- [ ] New docs, templates, or checklists replace repeated pain instead of adding ceremony.
