# Plan Review

Use before approving A1 work to become A2/A3 implementation.

- [ ] Agent inspected relevant code or sources.
- [ ] Plan names files likely to change.
- [ ] Scope matches the task brief and non-goals.
- [ ] Verification covers each acceptance criterion.
- [ ] Risks and stop conditions are explicit.
- [ ] No dependency, config, migration, auth/security, or broad refactor work is hidden in the plan.
- [ ] Human decisions are separated from agent actions.
