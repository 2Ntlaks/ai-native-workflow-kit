# Test Review

Use to decide whether evidence is strong enough for the task risk.

- [ ] Tests cover the acceptance criteria, not only implementation details.
- [ ] Bug fixes include a regression test or a documented manual check with a reason automated coverage was not added.
- [ ] Tests follow local framework and naming style.
- [ ] Commands and results are reported exactly.
- [ ] Manual checks include inputs, expected output, and observed output.
- [ ] Known gaps are listed as risks or follow-up work.
