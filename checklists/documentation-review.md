# Documentation Review

Use for docs changes and any implementation that changes user-visible behavior.

- [ ] Docs match current code, tests, and accepted decisions.
- [ ] New behavior, commands, or workflow changes are documented where users will look first.
- [ ] Stale or conflicting guidance was removed.
- [ ] Important claims are sourced or marked unverified.
- [ ] Links and referenced file paths are correct.
- [ ] The doc helps the human take an action, make a decision, or review work faster.
