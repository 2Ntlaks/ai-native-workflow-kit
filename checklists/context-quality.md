# Context Quality

Use before the agent starts reading or editing. The context should reduce guessing, not flood the thread.

- [ ] Current repo paths or source links are provided instead of pasted noise.
- [ ] Exact error text, logs, or reproduction steps are included when relevant.
- [ ] Current behavior and expected behavior are both stated.
- [ ] Assumptions are labeled.
- [ ] Sensitive data is removed or replaced with safe examples.
- [ ] Verification commands include expected evidence.
