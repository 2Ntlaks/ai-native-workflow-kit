# After Action Retrospective

## Purpose

Convert a completed task into a workflow improvement.

## When to Use

After a task had friction, defects, rework, or useful learning.

## When Not to Use

When nothing reusable was learned.

## Inputs Needed

- Review packet
- Human notes
- Failures or surprises

## Step-by-Step Process

1. Summarize what happened.
2. List what worked.
3. List what failed or slowed review.
4. Identify one reusable lesson.
5. Update a template, checklist, decision, or failure record if warranted.

## Expected Outputs

- Retrospective
- Optional kit update

## Acceptance Criteria

- Lesson is reusable.
- No blame or motivational filler.
- At most one small kit update is proposed by default.

## Example Prompt

```text
Write an after-action report for this task. Recommend at most one kit update.
```

## Common Failure Modes

- Writing a diary instead of a lesson.
- Updating too many docs.
- Not connecting failure to prevention.

## Review Checklist

- [ ] Is there a concrete prevention step?
- [ ] Does the update reduce future review time?
- [ ] Is the lesson reusable?
