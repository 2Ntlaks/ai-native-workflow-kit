# Contributing

Thanks for improving the AI-native workflow kit. Contributions should make it easier for a human to define, delegate, verify, review, or improve agent-assisted software work.

## What Belongs Here

- Practical workflows, templates, checklists, examples, and review gates.
- Source-backed claims about AI coding agents, tooling, verification, or human-in-the-loop workflows.
- Real examples from software work, especially C, Java, WebGL, databases, embedded systems, tutoring systems, and local-first desktop apps.
- Small improvements that reduce ambiguity, duplication, or review time.

## What Does Not Belong Here

- Motivational AI content.
- Prompt collections without workflow context.
- Tool-specific claims that are not marked `verify before use`.
- Broad rewrites that duplicate existing docs.
- New process for its own sake.

## Contribution Checklist

- [ ] The change helps a human make a decision or review agent work faster.
- [ ] The change is scoped to the smallest useful improvement.
- [ ] Important claims are added to `research/claims-index.md` or a source card.
- [ ] Tool-specific behavior is kept in `adapters/`.
- [ ] Examples are realistic enough to hand to an agent.
- [ ] Duplicated or vague nearby text was removed instead of expanded.

## Review Standard

Prefer direct, reviewable edits. A good contribution should answer:

- What task does this help with?
- What human decision or review step gets easier?
- What source supports any important claim?
- What should an agent stop and ask before doing?
