# Anti-Overengineering

Use this file when a task starts producing more process than evidence. The fix is usually to reduce scope, delete repeated text, or choose a smaller workflow.

## Keep It Small

- Use the smallest workflow that still controls the task risk.
- Use one agent unless independent research tracks are clearly cheaper.
- Add a template only after the same task shape repeats.
- Delete guidance that does not change what the human or agent will do.

## Remove Or Rewrite When

- The sentence would apply to almost any project.
- The advice cannot be checked in a review.
- The same rule appears in a more specific file.
- The text encourages confidence without asking for evidence.
