# Operating Model

The operating loop is: define outcome, choose workflow, choose autonomy, prepare context, ask for research or plan, approve scope, implement, verify, review, and learn.

## Roles

The human is product owner, permission gate, reviewer, and final decision-maker. The agent is researcher, planner, implementer, tester, documenter, and reviewer only within the selected autonomy level.

## Default Autonomy

Use A1 for unclear work and A2 for bounded implementation. Use A3 only when the workflow is understood and verification is strong. A4 and A5 are exceptional.
