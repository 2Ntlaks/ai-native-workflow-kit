# Glossary

## Acceptance Criteria
Observable conditions that must be true for the task to be accepted.

## Agent
An AI coding system that can use context and tools to perform software work.

## Autonomy Level
A chosen boundary for how much the agent may decide or do before asking the human.

## Context Packet
The task-specific bundle of objective, files, constraints, commands, and acceptance criteria.

## Diff Inspection
Human review of exactly what changed, independent of the agent's summary.

## Evidence Of Correctness
Test output, build output, screenshots, logs, or reasoning that supports the claim that the change works.

## Review Packet
The final task report containing summary, files changed, decisions, tests, evidence, risks, follow-up, and review focus.
